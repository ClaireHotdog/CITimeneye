Phase categories sit above phases in the hierarchy. Their primary power is as a bulk-audit tool: filter entries by category, then fix billing status in one action — instead of entry by entry.
Billable Categories
Billable — Client Deliverables
Monitor Products · Custom & Benchmarks · UX Research
Any phase producing a client-facing deliverable: reports, updates, matrices, benchmarks, presentations, readouts, client Q&A, quick questions.
Filter use: Confirm all entries are marked billable. Run hours-by-category to see total deliverable production time per client or monitor.
Billable — Research & Data
Monitor Products · Custom & Benchmarks · UX Research
Phases involving data gathering billed to clients: website monitoring, grading, desk research, competitive analysis, industry news, surveys, study fieldwork, LiGRE tagging, pace of change.
Filter use: Identify how much time goes into research vs. production. Useful for pricing future projects.
Billable — Client Meetings & Support
Monitor Products · Custom & Benchmarks · UX Research
Client-facing time billed directly: readout calls, screening meetings, presentation prep, client request responses, ad-hoc asks, UX Mind Melds.
Filter use: Track client-facing time separately from production time. Useful for TAP/account management reporting.
Non-Billable Categories
Non-Billable — Scoping & BD
Monitor Products · Custom & Benchmarks · All
Pre-contract and business development phases: annual planning, scoping calls, proposal development, pitch prep, demo prep, initial client conversations. Should never appear as billable.
Filter use: Flag any billable entries in this category as errors immediately. Track BD investment by team.
Non-Billable — Internal Operations
Internal Operations · All project types
Internal overhead not chargeable to clients: team meetings, 1:1s, all-hands, admin work, Timeneye maintenance, email management, status updates, hiring, performance reviews.
Filter use: Understand true overhead burden. Growing hours here may signal process inefficiency or understaffing.
Non-Billable — Product Maintenance
Monitor Products
Monitor upkeep not billed to clients: template updates, binder maintenance, tool/process improvements, post-project cleanup, consolidated document updating.
Filter use: Quantify invisible maintenance cost. Useful for justifying staffing or pricing adjustments on monitor renewals.
Non-Billable — Training & Development
All project types
Employee growth not chargeable to clients: onboarding, skill training, CI-wide trainings, security training, professional development, research methodology learning.
Filter use: Track investment in talent development. Useful for HR reporting and justifying L&D budgets.
Non-Billable — Marketing & BD Support
Internal Operations
Marketing and BD activities: blog posts, webinars, LinkedIn, conference prep, Pardot, social media, conference attendance, promotion efforts.
Filter use: Separate marketing investment from pure overhead. Track by campaign or conference to measure ROI.
Time Off
Time Off project only
All time-off phases: Vacation, Sick, Holiday, Lunch, Personal Day, Work Travel, Maternity/Paternity Leave, Bereavement, Jury Duty, Out of Office.
Filter use: Exclude from all client and operational reports. Useful for HR capacity planning when filtered separately.

⚡ Audit Workflow

  1. Go to Entries → filter by Non-Billable — Scoping & BD category
  2. Sort by Billing Status → look for any entries marked Billable
  3. Select all → bulk-update billing status to Non-Billable
  4. Repeat in reverse: filter Billable — Client Deliverables → look for Non-Billable entries → fix in bulk

📋 Tag Governance Rules

Require minimum 1 tag per entry (enable in Timeneye workspace settings) — only after lists are finalized.
Tags are entry-level only — they cannot be assigned to projects or phases (by design in Timeneye).
Restrict tag lists per project: Time Off project shows NO tags; Monitor projects show Deliverable Type + Work Channel + Contributing Team.
Do NOT replicate phase names as tags (e.g., don't create a "report" tag AND a "Report" phase — pick one).
Admin/Owner role maintains tag lists — regular members can select but not create new tags.
Review tag usage quarterly; retire unused tags to prevent creep.
Color-code projects by type so employees can instantly navigate to the right project. Within each color family, use lighter shades for sub-projects and darker for the primary monitor or project.

🎨 Implementation Tips
Set colors when creating or migrating 2026 projects — don't wait until all projects exist.
Favorite pinned projects appear at top of list; color + favorite = fastest possible navigation for regular users.
Lighter shades within a color family can distinguish sub-types (e.g., lighter blue for client request sub-projects under a monitor).
Archived projects lose visibility in the main list — archive 2025 projects after migration to keep the color-coded list clean.
Add a project description (Timeneye feature) alongside color so employees see a tooltip explaining what to log there.
